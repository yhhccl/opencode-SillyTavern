# 角色卡与条目编写铁律

本文是 `角色卡编写指南` 的提炼，AI 在创建世界书条目时必须遵守。

---

## 一、角色基础信息结构

角色条目分四个部分，**不多不少**：

```
角色档案:
  基本信息:    姓名、年龄、性别、身份、与{{user}}关系
  外貌特征:    只写偏离AI数据库默认值的特征
  背景设定:    只写改变了这个人的关键事件
  关系设定:    具体的互动画面，不写抽象形容
```

### 铁律：性格不写在这里

"她17岁，高二，吉他手"是基本信息。"她热情、叛逆"是性格——后者有独立条目。

混在一起 = AI 读到基本信息就开始调用性格标签演戏，后面的性格条目全在打架。

---

## 二、外貌只写特征，不写美感

**核心原则：AI 有自己的数据库。你只写偏离数据库默认认知的部分。**

| 角色类型 | AI 默认就知道 | 不用写 |
|----------|-------------|--------|
| 中国人 | 黑发黑眼黄皮肤 | ✓ 不用 |
| 日本人 | 黑发 | ✓ 不用 |
| 精灵 | 尖耳 | ✓ 不用 |
| 18岁女生 | 年轻、皮肤好 | ✓ 不用 |

**写什么：** 异色瞳、疤痕、纹身、义肢、特殊发色、标志性校服、体型显著偏差、习惯性饰品

**不写什么：** 万能美人描写（"精致""白皙""好看"）、意象比喻（"晨光般暖白"）、默认值

**判断标准：遮住名字，能不能只靠外貌特征认出是谁？能→好，不能→废话。**

### 错误 vs 正确

| 错误（废话） | 正确（特征） |
|-------------|------------|
| 精致的脸蛋，白皙的皮肤，桃花眼 | 淡蓝色瞳（Waardenburg综合征） |
| 身材匀称，亭亭玉立 | 一米六八，在女生里偏高 |
| 温婉优雅 | 左手食指和中指有弹吉他磨出的茧 |

---

## 三、背景只写关键事件

**只写"让这个角色变成现在这个样子"的事。** 不是人生年表。

写什么：家庭背景（有影响的）、经济状况（有影响的）、关键经历（改变角色的）、社交关系
不写什么：每个年龄段的流水账、无关童年琐事、"她小时候很可爱"类废话

**测试：删掉这条背景，角色会不会变？不会→删。**

---

## 四、关系写具体画面

不写"他们之间有着深厚的感情"→写具体做了什么。
不写"从小一起长大关系很好"→写"有记忆起就在一起"。

---

## 五、世界观：用最少的字说清所有设定

世界观不是小说设定集，是提示词。它**常驻**，每轮都在吃 token。

### 三种世界类型

| 类型 | AI 认知 | 写法 |
|------|---------|------|
| A 真实背景 | 全知道（现代东京、唐朝） | 只写 AI 不知道的虚构部分，1个条目 |
| B 小世界 | 知道套路（剑与魔法、赛博朋克） | 只写和默认套路不同的地方，1-3条目 |
| C 大世界 | 完全不知道 | 分层：总纲/速览常驻 + 详情按需加载 |

### 压缩原则

所有"是一个""存在着""分别是"等连接词 → 用冒号/逗号/列举替代。

| 冗余 | 压缩 |
|------|------|
| 这是一个剑与魔法的世界 | 剑与魔法世界观 |
| 修炼者需要通过不断的修炼来提升自己的境界 | 修炼提升境界 |
| 世界由三大帝国统治，分别是... | 三大帝国：A、B、C |

**不写：** 主观评价（"强大的帝国"）、比喻（"如同天堑的裂谷"）、废话（"充满机遇与挑战的世界"）、AI 已知信息

**压缩≠删除：** 另一个不了解你世界的 AI，只看这份条目能不能完整还原设定？能→对了。

## 六、世界书配置铁律

### 位置规则

| 内容类型 | 位置 | flag |
|----------|------|------|
| 世界观总纲、背景、社会规则 | ↑Char (0) | `--position 0` |
| 角色速览（多角色卡） | ↑Char (0) | `--position 0` |
| 角色详细信息 | ↓Char (1) | `--position 1` |
| NPC 详情 | ↓Char (1) | `--position 1` |
| 场景/事件/物品 | ↓Char (1) | `--position 1` |
| 二次解释（行为纠正） | @D depth=0 (4) | `--position 4 --depth 0 --role 0` |
| **D1+ 深度** | **绝对禁止** | 破坏聊天记录完整性 |

### 蓝灯 vs 绿灯

| 卡片类型 | 激活策略 |
|----------|----------|
| **单角色卡** | **全部蓝灯**（`--constant`）。拆成10个条目也全部蓝灯。这是铁律，不可改绿灯 |
| **多角色卡** | 速览蓝灯 + 各角色详细绿灯（`--keys 角色名,昵称`） |
| 世界观总纲 | 蓝灯 `--constant` |
| NPC/场景 | 绿灯 `--keys ...` |
| EJS 控制器 | 蓝灯；被控制的条目 → 禁用 |

### 递归设置

**所有条目必须 `--prevent-recursion`。** 无一例外。不勾的话，关键词连锁触发 → token 爆炸。

### 关键词格式

- **用英文逗号** `,` 分隔 — 中文逗号 `，` 直接失效
- 覆盖所有称呼：全名、昵称、外号、职务
- 示例：`--keys "林小雨,小雨,班长"`

### 扫描深度

绿灯条目加 `--scan-depth 2`，只看最后2条消息。

### 顺序

同位置内，数字越大越靠后：
- 世界观总纲 order=1
- 背景设定 order=2-3
- 角色速览 order=4
- 场景/事件 order=50-98
- 角色详细信息 order=99
- NPC order=100
