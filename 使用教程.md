# OpenCode SillyTavern 使用教程

这是一份面向日常使用的快速教程，适合第一次上手和后续查命令。

## 阅读建议

如果你只是想把项目跑起来并开始使用，只看这份教程就够了。

下面这些文档可以先跳过：

- `PROJECT_SPEC.md`：开发规划和项目规格
- `前端角色卡世界书改造执行方案.md`：历史改造方案
- `PROJECT_STRUCTURE.md`：仓库结构说明

等你需要二次开发、整理仓库或排查底层逻辑时，再回来看这些文档会更合适。

## 1. 项目是做什么的

这个项目把几部分拼在了一起：

- `OpenCode / Codex` 负责对话与角色扮演
- `web-frontend/` 提供网页聊天界面
- `角色卡/` 保存角色卡、世界书和每张卡的独立数据
- `scripts/novelai-generate.py` 提供 NovelAI 生图能力
- `airp-sillytavern/` 负责上下文整理和世界书激活

你可以把它理解成一个本地版的角色扮演前端：

- 能聊天
- 能切换角色卡
- 能编辑世界书
- 能查看上下文
- 能异步生图

## 2. 首次准备

### 环境要求

- Python 3.8 或更高
- 已安装 OpenCode / Codex
- 如果要生图，需要 NovelAI API Key

### 环境变量

根目录有一个 `.env.example`，复制为 `.env`，然后填入你的密钥：

```env
NOVELAI_API_KEY=你的密钥
```

## 3. 项目目录怎么理解

最常用的目录如下：

```text
web-frontend/           网页前端和本地服务
角色卡/                  角色卡目录
角色卡/example-card/    示例角色卡
scripts/                生图和标签提取脚本
skills/                 文风和 RP 规则
airp-sillytavern/       上下文构建与世界书激活
memory/reference.md     全局参考记忆
```

每张角色卡现在是一个独立目录，例如：

```text
角色卡/example-card/
├── card.json
└── worldbooks/
    └── main.json
```

运行时数据不会建议提交到 GitHub，比如聊天记录、生图结果、AIRP 会话缓存，这些已经在 `.gitignore` 里排除了。

## 4. 启动方式

### 方式 A：网页前端，推荐

在项目根目录运行：

```powershell
python web-frontend/server.py
```

然后浏览器打开：

```text
http://127.0.0.1:8765/index.html
```

打开后你可以：

- 直接聊天
- 在顶部切换角色卡
- 在设置面板里编辑角色卡
- 编辑世界书
- 查看上下文
- 点击消息里的“生成插图”

### 方式 B：终端配合 OpenCode 指令

如果你是按 OpenCode 的工作流来用，可以在 OpenCode 里使用这些命令：

```text
/play example-card
/cards
/switch <card>
/resume
/save
```

## 5. 角色卡怎么用

### 启动角色卡

首次启动示例卡：

```text
/play example-card
```

如果你在网页里用，当前角色会直接体现在顶部切换器里。

### 新增角色卡

在 `角色卡/` 下新建一个目录，例如：

```text
角色卡/my-card/
```

里面至少放：

```text
角色卡/my-card/card.json
角色卡/my-card/worldbooks/main.json
```

最简单的做法是复制 `角色卡/example-card/`，再改内容。

### 前端编辑角色卡

打开网页右上角设置面板后，可以编辑这些字段：

- 角色名称
- 标签
- 角色描述
- 性格
- 场景
- 开场白
- 示例对话
- system prompt
- post history instructions

改完点“保存角色卡”即可。

## 6. 世界书怎么用

世界书在每张角色卡目录下：

```text
角色卡/<card>/worldbooks/main.json
```

网页前端已经支持：

- 世界书列表读取
- 条目切换
- 新增条目
- 删除条目
- 编辑关键词、内容、位置、顺序
- 触发预览

### 世界书触发预览

在设置面板里输入一段文本，点击“触发预览”，可以看到：

- 哪些条目会激活
- 激活原因是什么

这个功能适合检查关键词是否写对。

## 7. 聊天怎么工作

网页聊天的流程是：

1. 你输入消息
2. 前端调用 `/api/submit`
3. 服务端构建 AIRP 上下文
4. 进入等待回复状态
5. 回复写入后，聊天区自动刷新

当前聊天页已经支持：

- AI 头像靠左，用户头像靠右
- 气泡居中分布
- 移动端风格输入区
- 空输入时发送按钮置灰
- 长文本自动换行
- 消息中的插图圆角显示

## 8. AIRP 上下文怎么看

前端设置面板里有“上下文检查”区域，可以看到：

- 当前阶段
- 当前角色卡
- 最近一次用户输入
- 上下文预算
- 历史保留轮次
- 激活的世界书

这个区域适合排查：

- 为什么角色没按预期说话
- 为什么世界书没触发
- 为什么上下文被裁剪

## 9. 生图怎么用

### 前端里直接生图

如果一条 AI 消息里带有 `[img: ...]` 标签，前端会显示“生成插图”按钮。

点击后会进入异步队列：

- `queued` 排队中
- `running` 生成中
- `done` 已完成
- `failed` 失败

生成成功后图片会直接显示在该消息下面。

### 脚本方式生图

也可以直接运行脚本：

```powershell
python scripts/novelai-generate.py -p "1girl, library, masterpiece"
```

或者从 RP 日志里提取标签后生成：

```powershell
python scripts/extract-img.py 角色卡/example-card/rp-log.txt -g --latest-only
```

## 11. 常见问题

### 页面打不开

先确认服务有没有启动：

```powershell
python web-frontend/server.py
```

然后看地址是不是：

```text
http://127.0.0.1:8765/index.html
```

### 生图失败

优先检查：

- `.env` 是否存在
- `NOVELAI_API_KEY` 是否填写正确
- 网络是否可访问 NovelAI

### 世界书不触发

先用前端里的“触发预览”测试关键词。

如果预览也不触发，通常是：

- 关键词没写对
- 条目被禁用
- 次关键词过滤条件不满足

### 角色卡切换后内容不对

先确认当前顶部选择的是哪张卡。

如果还是不对，重新切换一次角色卡，或者刷新页面，让前端重新加载当前卡和世界书。

## 12. 推荐使用流程

如果你是日常使用，最省心的一套流程是：

1. 配好 `.env`
2. 运行 `python web-frontend/server.py`
3. 打开 `http://127.0.0.1:8765/index.html`
4. 先用 `example-card` 跑通
5. 再复制示例卡创建自己的角色卡
6. 在前端里改角色卡和世界书
7. 对话中按需点击“生成插图”
