# OpenCode RP — 游玩指南

> 本文档涵盖从零开始到熟练游玩的所有步骤。适合首次使用者。

---

## 一、快速开始

### 环境要求

- **OpenCode CLI** ≥ 1.14（[安装](https://opencode.ai)）
- **Python** ≥ 3.8
- **NovelAI API Key**（生图需要，纯文本扮演可选）

### 首次准备（仅一次）

```
1. 配置 .env: NOVELAI_API_KEY=你的密钥
2. 放角色卡到 角色卡/ 目录
3. 确认 opencode 可用: opencode --version
```

### 三步开始扮演

```
步骤 1 — 终端打开项目目录，启动 opencode（固定端口）:
  cd "C:\Users\hu\Desktop\opencode SillyTavern"
  opencode --port 4096

步骤 2 — 在 opencode TUI 中加载角色并进入 Web 模式:
  /play example-card
  /rp

步骤 3 — 浏览器打开 http://localhost:8765
  输入对话 → AI 自动回复
```

## 二、Web 前端模式

### 数据流

```
┌─ 浏览器 :8765 ────────────────────────────────────┐
│  微信风格聊天 UI                                    │
│  输入 → POST /api/submit → 1.5s 轮询 /api/content   │
│  右侧绿色气泡 = 你，左侧白色气泡 = AI               │
└──────────────┬────────────────────────────────────┘
               │ HTTP
┌─ server.py :8765 ─────────────────────────────────┐
│  poller 线程: .pending → 读 web-input.txt             │
│  → 调用 opencode_client.send_message()               │
│  → POST http://127.0.0.1:4096/tui/clear-prompt       │
│  → POST /tui/append-prompt {"text":"【Web 前端 RP】\n用户输入: ...\n..."}
│  → POST /tui/submit-prompt                            │
└──────────────┬────────────────────────────────────┘
               │ TUI REST API
┌─ opencode TUI :4096 ──────────────────────────────┐
│  消息注入输入框 → AI 按 AGENTS.md 规则处理           │
│  → 读角色卡 → 读聊天历史 → 生成 200-1000 字叙事       │
│  → 提取 [img: ...] 标签                              │
│  → Write 追加 rp-log.txt + 更新 session-state.md       │
└──────────────┬────────────────────────────────────┘
               │ 文件监控
      rp-log.txt 增长 → 提取叙事 → 返回 poller
               │
      poller: append_message → content.js
      → 浏览器 1.5s 轮询 /api/content → DOM 刷新
```

### 输入/输出分离

| 角色 | 来源 | 前端渲染 |
|------|------|----------|
| **用户输入** | `web-input.txt`（前端 POST 原始消息，永不修改） | 右侧绿色气泡 + "我"头像 |
| **AI 输出** | `rp-log.txt`（AI 按 AGENTS.md 规则生成的叙事正文） | 左侧白色气泡 + "角"头像 + 🎨插图按钮 |

### 前端界面

- **聊天区**：微信风格气泡，消息带时间戳
- **输入框**：Enter 发送，Shift+Enter 换行
- **状态栏**：发送后显示 "AI 思考中..."
- **主题切换**：点击 `◐` 切换亮色/暗色
- **设置面板**：点击右上角 `⋯` → 角色卡编辑、世界书管理、上下文检查
- **插图按钮**：AI 回复下方的 `🎨 生成插图` → 调用 NovelAI 生图

### 启动 server.py 的三种方式

| 方式 | 说明 |
|------|------|
| `/rp`（推荐） | 在 opencode TUI 中输入，AI 自动启动 server.py |
| `python web-frontend/server.py` | 手动在终端启动 |
| `退出RP` | 在 TUI 中停止 server.py，恢复编辑模式 |

## 三、终端模式（不用 Web）

如果不需要 Web 界面，直接在 opencode TUI 中扮演：

```
1. opencode（不限端口号）
2. /play <角色卡>
3. 直接输入对话 → AI 生成叙事回复
```

**终端模式与 Web 模式可随时切换：**
- 终端模式中输入 `/rp` → 启动 Web 模式
- Web 模式中输入 `退出RP` → 回到终端模式

**每轮自动流程：**
```
你输入 → AI 写叙事（*动作用斜体*，对话不加引号）
  → AI 回扫正文提取 [img: ...] 标签
  → Write 追加到 rp-log.txt
  → Write 更新 session-state.md
  → Write 更新 memory/project.md
```

## 四、角色卡管理

### 核心命令

| 命令 | 效果 | 示例 |
|------|------|------|
| `/play <card>` | 启动或切换角色卡 | `/play 巨根症` |
| `/cards` | 列出所有可用角色卡 | |
| `/switch <card>` | 保存当前进度后切换 | `/switch card-b` |
| `/resume` | 恢复上次会话（新窗口） | |
| `/save` | 手动保存状态快照 | |

### /play 流程

```
/play 巨根症 →
  1. 若当前有其他角色卡 → 自动保存状态
  2. 写 current-card.txt = "巨根症"
  3. 检查 角色卡/巨根症/ 目录:
     存在 → 读 session-state.md + rp-log.txt 末30行 → 接续
     不存在 → 读 card.json → 初始化存档 → 从 first_mes 开场
```

### 卡片目录结构

```
角色卡/{card}/
├── card.json            ← 角色卡本体（名称、描述、开场白等）
├── worldbooks/           ← 世界书（main.json 等）
├── rp-log.txt            ← 对话完整记录
├── session-state.md      ← 场景状态快照
├── generated/            ← NAI 生图输出
└── memory/
    ├── project.md        ← 剧情进度摘要（≤300字）
    ├── feedback.md       ← 用户反馈
    └── user.md           ← 用户设定
```

### 恢复上次游玩

```
新终端:
  1. cd 项目目录
  2. /play 巨根症
  → AI 自动: 读 session-state.md → 读 rp-log.txt 末30行 → 自然接续
  3. /rp → 继续 Web 模式
```

## 五、生图（NovelAI）

### 自动标签提取

AI 每轮叙事后自动提取 NAI 标签：

```
*她推开木门，午后的光从窗格漏进来，在旧书脊上切出一道道金线...*

[img: 1girl, librarian, glasses, afternoon light, old library, bookshelves, dust motes, warm atmosphere, masterpiece, best quality]
```

### 生图命令

| 命令 | 效果 |
|------|------|
| `/img 生成` | 生成最新一张 [img:...] 标签对应的图 |
| `/img 全部` | 生成所有未生成的标签 |
| `/img 横版` | 同上，横版 1216×832 |
| `/img furry` | 使用 Furry 模型 |

### 前端生图

点击 AI 回复下方的 `🎨 生成插图` 按钮，异步生图（约 60-120 秒），完成后自动出现在气泡下方。

### 手动生图

```bash
# 提取 + 生成
python scripts/extract-img.py 角色卡/{card}/rp-log.txt -g --latest-only

# 直接调用
python scripts/novelai-generate.py -p "1girl, black hair, school uniform, masterpiece, best quality"
```

## 六、世界书

### Web 设置面板

浏览器右上角 `⋯` → 世界书：

- **选择世界书文件** — 每个角色卡可有多本世界书（main.json 等）
- **条目管理** — 新增、编辑、删除世界书条目
- **触发预览** — 输入文本测试哪些条目会被激活
- **保存** — 自动同步到 `角色卡/{card}/worldbooks/`

### 条目字段

| 字段 | 说明 |
|------|------|
| 主关键词 | 逗号分隔，匹配时触发 |
| 次关键词 | 配合 selective 使用 |
| 内容 | 激活时注入的文本 |
| 位置 | 前置（0）或后置（1） |
| 常驻 | 始终激活 |
| 禁用 | 临时关闭此条目 |

## 十、FAQ

### Q: 浏览器输入后没有回复？

**A:** 检查 3 点：
1. opencode 是否以 `--port 4096` 启动
2. server.py 是否在运行（检查 `web-frontend/server.pid`）
3. 角色卡是否已加载（检查 `current-card.txt`）

### Q: 生图很久没反应？

**A:** NovelAI V4.5 生成约 60-120 秒。超时 180 秒会报错，重试即可。

### Q: 消息出现乱序或最后一条不显示？

**A:** 已修复（v2）。`content.js` 使用原子写入，前端有内容校验 + 3 次重试。如仍出现，F5 刷新浏览器。

### Q: 怎么改角色卡内容？

**A:** 两种方式：
1. Web 设置面板（浏览器右上角 `⋯`）
2. 退出 RP 模式后在 opencode TUI 中用 Write 工具直接编辑 `card.json`

### Q: 能同时玩多张角色卡吗？

**A:** 可以。`/switch <card>` 保存当前卡状态后切换。每张卡独立存档，互不干扰。

### Q: 换电脑了怎么继续？

**A:** 
1. 新电脑装好 opencode + Python，复制项目目录
2. `/play <角色卡名>` → AI 自动恢复

### Q: 想跳过这轮的 [img:...] 标签？

**A:** 对话中 AI 只在"有画面感的时刻"自动输出标签，平淡对话自动跳过。如需强制跳过，说"这轮不需要生图"。

### Q: server.py 启动失败？

**A:** 
1. 确认 Python 版本 ≥ 3.8
2. 确认路径中没有他人占用端口 8765
3. 查看 `web-frontend/server-stderr.log`
4. 删除 `web-frontend/__pycache__` 后重试
